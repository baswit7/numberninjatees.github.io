Pinterest API Use Case — NumberNinjaTees Marketing OS

A transparent description of the intended Pinterest integration for the NumberNinjaTees apparel brand, owned by Bas Wit in IJmuiden, The Netherlands.

APP: NumberNinjaTees Marketing OS BRAND: NumberNinjaTees SALES: ETSY USE: OWN BRAND MARKETING

Exact App DescriptionApplication Purpose

NumberNinjaTees Marketing OS is an automation and analytics platform for managing apparel marketing campaigns for the NumberNinjaTees brand.

The platform helps create product pins, manage Pinterest boards, optimize product descriptions, track content performance and improve marketing workflows for data-focused apparel products targeted at data analysts, programmers, developers, Excel users and tech enthusiasts.

The Pinterest API is used only for authorized NumberNinjaTees accounts through OAuth. The app does not scrape Pinterest, does not access accounts without permission, does not sell Pinterest data and does not post spam. API access is used for creating and managing the brand’s own pins and boards and for analyzing performance of the brand’s own Pinterest content.

Authorized ActivityAPI Actions

The integration is intended only for NumberNinjaTees brand marketing. With authorized access, it may:

  • Read the brand's own Pinterest boards.
  • Create and publish product Pins for the brand's own apparel listings only after intentional operator approval.
  • Manage the brand's own Pins and boards.
  • Analyze performance of the brand's own Pinterest content.

SafeguardsProhibited Activity

NumberNinjaTees Marketing OS is not designed or permitted to:

  • Scrape Pinterest content or bypass Pinterest APIs.
  • Post spam or create misleading engagement.
  • Sell or share Pinterest data for third-party marketing.
  • Access an account without OAuth authorization.
  • Expose tokens, client secrets or credentials publicly.
  • Automatically publish a Pin without the authorized operator selecting and approving that content.

Permission RequestPossible Scopes

The app may request only scopes required for its authorized brand workflow:

pins:readpins:writeboards:readboards:writeuser_accounts:readanalytics:read

analytics:read is requested only if available and necessary for approved brand-content analysis. Final requested permissions will follow Pinterest's available scopes and minimum-access principle.

AuthorizationOAuth Redirect Flow

  1. The authorized NumberNinjaTees operator starts a Pinterest connection.
  2. Pinterest presents its OAuth authorization screen and requested scopes.
  3. After approval, Pinterest redirects to the registered callback URL.
  4. Secrets and tokens are never displayed on public pages or committed to this site repository.
  5. Authorized actions apply only to NumberNinjaTees marketing content.

Redirect URL for review:
